ABSTRACT

In our fast-paced daily lives, managing tasks effectively is essential for productivity. The "Advanced To-Do Web Application" aims to provide an intuitive and efficient way for users to record, organize, and monitor their day-to-day activities.

Unlike basic to-do lists, this application is a full-stack web project built using Python's Flask framework for the backend, and HTML, CSS, and Vanilla JavaScript for the frontend. The application features a robust authentication system allowing multiple users to register, log in, and securely manage their own tasks. Data is persistently stored using a relational SQLite database via the SQLAlchemy Object Relational Mapper (ORM).

Users can add tasks with priorities, assign due dates, categorize tasks, and mark them as complete. The seamless RESTful API architecture ensures smooth data transactions between the client UI and the server.

1. INTRODUCTION

A To-Do List application is a complex piece of software that can be used to manage personal and professional workloads. The primary objective is to keep track of tasks that need to be accomplished, ensuring nothing is forgotten. While traditional pen and paper methods are helpful, they lack the dynamism of modern software—such as reminders, categories, priorities, and accessibility from multiple devices.

This project modernizes task management by bringing it to the web. It uses a secure authentication system, ensuring user data remains private. The web application allows seamless CRUD (Create, Read, Update, Delete) operations, making it highly interactive.

2. REVIEW OF LITERATURE

Existing task management tools range from simple sticky notes applications to complex Kanban boards. Common solutions like Microsoft To Do or Todoist offer robust features but are proprietary. Developing a custom web-based solution allows for understanding the core mechanisms of web programming, REST APIs, and database interactions.

By studying related web apps, this project implements necessary features like Due Date tracking and Priority flags, offering all standard tools in a minimalist interface.

5. METHODOLOGY & IMPLEMENTATION

The system was built iteratively. Phase 1 involved database schema design using SQLAlchemy models. Phase 2 handled the backend Flask routes for User Registration/Login and To-Do CRUD APIs. Phase 3 focused on creating the frontend UI templates to communicate with the APIs.

6. DATABASE DESIGN

The system uses relational tables linked by foreign keys (User ID).

Table: User

  • id: (INTEGER, Primary Key)
  • email: (VARCHAR, Unique)
  • password_hash: (VARCHAR)

Table: Todo

  • id: (INTEGER, Primary Key)
  • title: (VARCHAR)
  • completed: (BOOLEAN)
  • priority: (VARCHAR: low, medium, high)
  • user_id: (INTEGER, Foreign Key to User.id)

7. SYSTEM TESTING

The system underwent rigorous boundary testing. Manual testing cases included:

  • Authentication Testing: Verified that unauthorized users get redirected to the login page when trying to access /api/todos.
  • CRUD Operations: Verified tasks can be accurately added, modified, and deleted without page refreshes.
  • Database Integrity: Verified foreign key constraints prevent orphaned task rows when an account is managed.

8. CONCLUSION & FUTURE SCOPE

The project successfully demonstrates the integration of full-stack logic. The ORM effectively mitigates SQL Injection vulnerabilities found in standard applications. In the future, this application can be scaled to include Push Notifications, Multi-user Task Sharing, and real-time collaboration features to rival commercial task-manager products.

Models Definition Example (SQLAlchemy)

class Todo(db.Model):
    id = db.Column(db.Integer, primary_key=True)
    title = db.Column(db.String(100), nullable=False)
    completed = db.Column(db.Boolean, default=False)
    user_id = db.Column(db.Integer, db.ForeignKey('user.id'))
